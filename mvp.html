Understanding "Thin Slice" vs "MVP" in This RFP
🍕 "Thin Slice" — the project selection

Sri Lanka's DHP blueprint has 17 components total. This RFP selects 5 core systems (NEHR, NHDX, IdP, Terminology, e-Referral) plus cross-cutting modules (Patient Access, Common Requirements) for initial delivery — hence "thin slice" of the full platform. This is the procurement scope decision at the programme level, not an MVP definition.

🚀 "MVP" — Deliverable 2, Week ~14

All 5 selected systems must be delivered in a basic but working form by Week ~14. The RFP does not define which individual requirements are in/out for MVP — that is proposed by the vendor in the Inception Report (Week 4) and agreed with MoH. ℹ §8 D1

Note on requirement-level MVP tags: The scoping in the tables and accordion below represents Allion Tech's recommended MVP scope for the Inception Report proposal — not something mandated by the RFP. The RFP intentionally leaves individual requirement prioritization to the vendor.

MVP vs Full Scope — Per System

Based on the Deliverable 2 description and overall RFP structure. Exact feature list is proposed in the Inception Report.

NEHR — National Electronic Health Record
MVP Scope (Target)
  • ✓ Core FHIR R4 data models (Patient, Encounter, Observation, Condition, MedicationRequest)

  • ✓ Basic read/write FHIR APIs for clinical encounters
  • ✓ Audit log generation for all access events
  • ✓ Basic Patient Access Module (view records via web)
  • ✓ Consent management foundation

  • → Advanced analytics (cohort analysis, trend, indicators)
  • → Bulk summary retrieval

✅ MVP Acceptance Criteria

Deliverable 2 (MVP) is accepted when UNOPS and MoH review and approve the submission. Payment of 20% of total contract value releases net 30 days after acceptance. The next deliverable (Connectathon, D3) cannot begin its 4-week countdown until D2 is accepted. ℹ §12 Payment

What Allion Tech should demonstrate
  • ✓ All systems deployable and running in a staging environment
  • ✓ Core FHIR APIs returning correct responses
  • ✓ Basic end-to-end flow: patient record creation → retrieval → referral
  • ✓ Automated CI/CD pipeline running builds and tests
  • ✓ Preliminary performance baseline results with synthetic data
  • ✓ Initial interoperability test scenarios documented
Risk: what can delay acceptance
  • ✗ MoH/UNOPS review cycles add buffer to the 10-week clock
  • ✗ Any scope agreed at Inception not met = rejection
  • ✗ If Inception Report acceptance is delayed, MVP clock shifts too
  • ✗ Delays here cascade directly to Connectathon (D3) and December 2026 hard stop
⏰
Hard External Deadline: December 2026

The HiQi project funding expires in December 2026. This is not a soft internal target — it is driven by an external grant from the Federal Government of Germany. Any delays in the MVP phase directly compress all remaining deliverables. ℹ §1.1
